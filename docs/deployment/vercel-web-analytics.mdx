# Getting Started with Vercel Web Analytics

This guide will help you get started with using Vercel Web Analytics on your Docusaurus project, showing you how to enable it, add the package to your project, deploy your app to Vercel, and view your data in the dashboard.

## Prerequisites

- A Vercel account. If you don't have one, you can [sign up for free](https://vercel.com/signup).
- A Vercel project. If you don't have one, you can [create a new project](https://vercel.com/new).
- The Vercel CLI installed. If you don't have it, you can install it using the following command:

```bash npm2yarn
npm install -g vercel
```

## Enable Web Analytics in Vercel

On the [Vercel dashboard](https://vercel.com/dashboard), select your Project and then click the **Analytics** tab and click **Enable** from the dialog.

:::note
Enabling Web Analytics will add new routes (scoped at `/_vercel/insights/*`) after your next deployment.
:::

## Add `@vercel/analytics` to your project

Using the package manager of your choice, add the `@vercel/analytics` package to your project:

```bash npm2yarn
npm install @vercel/analytics
```

## Add the Analytics Component to Your App

For Docusaurus, you need to create a custom Root component that wraps your entire application. This is the recommended approach for adding global components in Docusaurus.

### Step 1: Create the Root Component

Create a new file at `src/theme/Root.tsx`:

```tsx title="src/theme/Root.tsx"
import React from 'react';
import { Analytics } from '@vercel/analytics/react';

// Default implementation, that you can customize
export default function Root({ children }) {
  return (
    <>
      {children}
      <Analytics />
    </>
  );
}
```

If you're using TypeScript, you can also add proper types:

```tsx title="src/theme/Root.tsx"
import React, { ReactNode } from 'react';
import { Analytics } from '@vercel/analytics/react';

interface RootProps {
  children: ReactNode;
}

export default function Root({ children }: RootProps): ReactNode {
  return (
    <>
      {children}
      <Analytics />
    </>
  );
}
```

### Step 2: Verify the Integration

The Root component is automatically used by Docusaurus to wrap your entire application. The `Analytics` component from `@vercel/analytics/react` is a lightweight wrapper around the tracking script that provides seamless integration with React applications.

:::tip
The `@vercel/analytics/react` package automatically handles route changes in your Docusaurus site, so page views will be tracked as users navigate between documentation pages.
:::

## Deploy Your App to Vercel

Deploy your app using the following command:

```bash
vercel deploy
```

If you haven't already, we also recommend [connecting your project's Git repository](https://vercel.com/docs/git), which will enable Vercel to deploy your latest commits to main without terminal commands.

Once your app is deployed, it will start tracking visitors and page views.

:::note
If everything is set up properly, you should be able to see a Fetch/XHR request in your browser's Network tab from `/_vercel/insights/view` when you visit any page.
:::

## View Your Data in the Dashboard

Once your app is deployed, and users have visited your site, you can view your data in the dashboard.

To do so:

1. Go to your [Vercel dashboard](https://vercel.com/dashboard)
2. Select your project
3. Click the **Analytics** tab

After a few days of visitors, you'll be able to start exploring your data by viewing and [filtering](https://vercel.com/docs/analytics/filtering) the panels.

Users on Pro and Enterprise plans can also add [custom events](https://vercel.com/docs/analytics/custom-events) to their data to track user interactions such as button clicks, form submissions, or other custom actions.

## Advanced Configuration

### Custom Events

If you're on a Pro or Enterprise plan, you can track custom events. For example, to track when users click on a specific link:

```tsx
import { track } from '@vercel/analytics';

function MyComponent() {
  return (
    <button onClick={() => track('button_clicked', { label: 'Download' })}>
      Download
    </button>
  );
}
```

### Development Mode

By default, the Analytics component only tracks events in production. If you want to enable tracking in development mode for testing:

```tsx title="src/theme/Root.tsx"
import React from 'react';
import { Analytics } from '@vercel/analytics/react';

export default function Root({ children }) {
  return (
    <>
      {children}
      <Analytics mode="development" />
    </>
  );
}
```

:::warning
Remember to remove `mode="development"` before deploying to production, or set it conditionally based on the environment.
:::

### Debug Mode

To see debug information in the console:

```tsx title="src/theme/Root.tsx"
import React from 'react';
import { Analytics } from '@vercel/analytics/react';

export default function Root({ children }) {
  return (
    <>
      {children}
      <Analytics debug={true} />
    </>
  );
}
```

## Privacy and Compliance

Vercel Web Analytics is designed with privacy in mind:

- No cookies are used
- No personal data is collected
- GDPR and CCPA compliant
- All data is anonymized

Learn more about how Vercel supports [privacy and data compliance standards](https://vercel.com/docs/analytics/privacy-policy) with Vercel Web Analytics.

## Troubleshooting

### Analytics Not Showing Data

If you're not seeing any data in your analytics dashboard:

1. **Check that Analytics is enabled** - Make sure you've enabled Analytics in your Vercel project settings
2. **Verify deployment** - Ensure your latest deployment includes the Analytics component
3. **Check the network tab** - Look for requests to `/_vercel/insights/view` in your browser's developer tools
4. **Wait for data** - It may take a few hours for data to appear in the dashboard

### Build Errors

If you're getting build errors after adding the Analytics component:

1. **Check package installation** - Make sure `@vercel/analytics` is properly installed
2. **Verify TypeScript types** - If using TypeScript, ensure the package types are correctly resolved
3. **Check React version** - Ensure your React version is compatible (React 18+ recommended)
